# 📌 Project Overview

This project is a portfolio web application developed using Flask, HTML, CSS, and JavaScript. The purpose of the application is to showcase my projects, technical skills, and background as I transition from a clinical healthcare role into software development.

The application follows a structured multi-page layout, including Home, Projects, Project Details, About, and Contact sections, and demonstrates both front-end design and back-end functionality using Flask.

# 🚀 Key Features
🔹 Flask-Based Web Application: Built using the Flask framework with multiple routes, Each page is rendered dynamically using Jinja templates, Demonstrates understanding of GET and POST HTTP methods.

🔹 Dynamic Project Rendering: Project data is stored in a Python list of dictionaries, Projects are dynamically displayed using Jinja templating,This allows easy scalability by adding new projects without modifying HTML structure.

🔹 Project Filtering (JavaScript): Users can filter projects by category using interactive buttons, Implemented using JavaScript to dynamically show/hide project cards, Demonstrates DOM manipulation and event handling.

🔹 Suggestion Box (Flask POST Handling): Users can submit project suggestions via a form, Form data is handled using Flask’s POST method, Suggestions are stored in a Python list and displayed dynamically on the page, Demonstrates form handling, user interaction, and data rendering.

🔹 Contact Form: A functional contact form that accepts user input (name, email, message), Uses POST request handling in Flask, Displays a confirmation message upon submission, Input data is printed to the console for demonstration purposes.

# 🎨 Design Enhancements
🔹 Responsive Design: Implemented using CSS media queries, Layout adapts for desktop, tablet, and mobile devices, Navigation, hero section, and forms are optimized for smaller screens.

🔹 Modern UI Styling: Clean and minimal design with consistent color scheme, Use of gradients, shadows, and hover effects to enhance user experience, Buttons and cards include smooth transitions and animations.

🔹 Hero Section: Includes a profile image and introduction, Centrally aligned layout for better visual balance, Designed to create a strong first impression.

🔹 Dark Mode Toggle: Implemented using JavaScript, Allows users to switch between light and dark themes, Enhances accessibility and user preference.

# 💡 Additional Functionality
🔹 Smooth Scrolling: JavaScript is used to create smooth navigation between sections.

🔹 Interactive UI Elements: Hover effects on links and buttons improve user engagement, and cards animate slightly when hovered to create depth.

# 🧠 Code Structure & Best Practices
- The project follows a clear structure:
      app.py for Flask routes and logic
      templates/ for HTML files
      static/css/ for styling
      static/js/ for JavaScript functionality
- Jinja templating is used to avoid repetition and maintain clean code.
- Reusable components such as navigation and layout are handled using base.html.
